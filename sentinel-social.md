## LinkedIn

Excited to release ZeroPoint Sentinel v0.3 — multi-platform cryptographic governance for the network edge.

What started as a single-router experiment on an ASUS Merlin router in my barn is now a package that installs on any dnsmasq-based router, any Linux box, any Raspberry Pi, or any Docker host.

The big addition: mesh participation. Every Sentinel now generates an Ed25519 identity, announces its capabilities to ZeroPoint Core, and shows up as a first-class peer in the trust mesh. The same protocol that governs agent frameworks governs your router.

DNS filtering. Device monitoring. Anomaly detection. Blake3 hash-chained audit trails. Push notifications. All governed by policy. All provable.

One protocol. One graph. End to end.

Open source: https://github.com/zeropoint-foundation/zeropoint
Install: https://zeropoint.global/sentinel

## Hacker News title + comment

Title: ZeroPoint Sentinel: Governance-based network monitoring with mesh identity (multi-platform)

Comment: I built this to prove that cryptographic governance — Ed25519 identity, hash-chained audit trails, policy pipelines — works on constrained hardware. The Sentinel runs on a 32-bit ARM router with 256MB RAM. DNS filtering, device monitoring, anomaly detection, all flowing through a Guard → Policy → Audit pipeline.

v0.3 adds multi-platform support (Merlin, OpenWrt, Linux, Docker) and mesh participation — the router gets an Ed25519 keypair and announces itself to ZeroPoint Core as a first-class peer, same protocol as any other component in the mesh.
