Getting Started

Secure your compute space
in one command

ZeroPoint discovers your environment, wraps every interface with cryptographic governance, and fades into the background — guardian until needed. Every action produces a signed receipt. Every interface becomes accountable.

The CLI is fully functional. For the full visual experience, ZeroPoint launches a dashboard when appropriate.

Prerequisites

Install ZeroPoint

ZeroPoint is built in Rust. The install script handles the toolchain, builds from source, runs the test suite, and generates your cryptographic identity. On first launch, the Genesis ceremony establishes your node — a signed record of who you are and what governance rules are active.

$ curl -sSf https://install.zeropoint.global | bash

Requires: macOS or Linux · Rust toolchain (installed automatically) · ~5 minutes to build

What happens at Genesis

ZeroPoint generates an Ed25519 keypair, installs six governance gate rules (two constitutional, four operational), creates a tamper-evident audit chain, and writes a signed genesis record. This is your node's root of trust. Everything chains from here.

The Setup

Five phases to a secured compute space

After install, run zp secure to launch the adaptation wizard. Each phase is explained as it runs — what it does, why it matters, what changes. Accept the smart defaults or customize everything.

ZeroPoint begins by understanding your environment. It scans for shells, AI tools, running services, sensitive directories, and network interfaces. None of this is intrusive — ZeroPoint reads, it doesn't touch. The goal is to build a complete picture of your compute space and present it with a clear recommendation. You decide what gets wrapped.

Your shell is the foundation of compute space governance. ZeroPoint installs a lightweight preexec hook that evaluates commands before they execute. Safe commands pass through instantly — sub-millisecond overhead. Dangerous commands are blocked. Everything in between is logged with a signed receipt. You choose the posture.

permissive

Observe Only

Log everything but block nothing. Good for getting familiar with what ZeroPoint sees before enforcing policy.

balanced

Warn & Block Critical

Warn on risky operations, block critical threats like credential exfiltration and recursive system deletion. The right default for most users.

strict

Approve Everything

Require explicit approval for anything beyond a curated safe list. Maximum governance for high-security environments.

How MCP governance works

For Claude Code, ZeroPoint installs as an MCP server — it intercepts every tool call before execution and evaluates it against your policy set. The agent sees normal responses. Behind the scenes, every action is receipted, policy-checked, and chain-linked. If a call violates policy, it's blocked before the tool ever executes.

How PATH wrapping works

For other tools, ZeroPoint creates lightweight shims in ~/.zeropoint/bin/ which sits first in your PATH. When you (or an agent) invoke cursor, the shim evaluates the action through the guard, then passes through to the real binary. Same interface, now governed.

The outer perimeter. ZeroPoint watches your sensitive directories, governs Docker containers, and optionally proxies API calls. Not all of this needs to be active on day one. Sensitive directory watchers install by default — that's the minimum responsible posture. Everything else is available when you're ready.

After Setup

What's different now

zp status

Verify Anytime

One command shows every active governance layer — shell hooks, AI tool wrappers, file watchers, policy rules. If something's misconfigured, you'll know immediately.

~0.5ms overhead

Invisible Governance

The guard evaluates locally with sub-millisecond latency. No network round-trip, no waiting. Safe commands pass through before you notice. Governance that disappears into the workflow.

zp secure --wizard

Reconfigure Without Starting Over

Change your posture, add a new shell, wrap a new AI tool, or adjust which directories are watched. The wizard picks up where you left off.

localhost:3000

Visual Dashboard

For complex environments, launch the dashboard to see your governance surface visualized — policy decisions, receipt chains, AI tool activity, all in one place.
