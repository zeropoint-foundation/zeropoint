ARCHITECTURE & DATA STRUCTURES
================================================================================

Configuration System:
- DNSConfig: log_path, blocklist_urls, blocklist_cache_path, rate_limit_qps
- DeviceConfig: dhcp_lease_file, mac_blocklist, rate_limit_connections_per_minute
- AuditConfig: db_path, max_entries_memory
- ServiceConfig: pid_file, log_file, log_level, monitor_interval, zp_server_url

Decision Pipeline (Guard → Policy → Audit):
1. Guard: Blocklist checking + rate limiting (token bucket)
2. PolicyEngine: Rule-based policy evaluation
3. AuditStore: Blake3 hash-chained SQLite records

Core Data Structures:
- PolicyDecision enum: ALLOW, BLOCK, WARN, REVIEW, SANITIZE
- PolicyContext: action, trust_tier, channel
- GateResult: decision, risk_level, trust_tier, audit_entry, applied_rules
- AuditEntry: id, timestamp, prev_hash, entry_hash, actor, action, decision
- TrustTier enum: TRUSTED, UNTRUSTED, UNKNOWN, SUSPICIOUS, BLOCKED

QUICK START
================================================================================

1. Installation:
   $ cd /opt/etc/zp-sentinel
   $ chmod +x install.sh
   $ ./install.sh

2. Enable DNS Logging:
   $ echo "log-queries=extra" >> /jffs/configs/dnsmasq.conf.add
   $ service restart_dnsmasq

3. Start Service:
   $ /opt/etc/init.d/S99zp-sentinel start

4. Monitor:
   $ tail -f /opt/var/log/zp-sentinel.log
   $ /opt/bin/zp-sentinel status
   $ /opt/bin/zp-sentinel audit

5. Block Devices:
   $ /opt/bin/zp-sentinel devices
   $ /opt/bin/zp-sentinel block 38:a5:c9:20:4a:a5

6. Verify Integrity:
   $ /opt/bin/zp-sentinel verify
